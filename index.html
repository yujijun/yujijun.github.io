如何总结和整理学术文献

发表于 2021-03-08 | 分类于 锦囊

  如果说科研是一场学术竞赛，那么文献就是我们的课本，阅读笔记就是课堂笔记，写自己的学术论文就是一次考试。 如何选自己的学术课本，如何阅读和整理的自己的课堂笔记，将很大程度上决定自己在本次学术考试中的成败。可以这样说，高效的阅读学术论文，是科研人员最基本也是最重要的能力之一。
  下面就根据自己的经验，谈谈如何总结和整理自己的学术文献。

总结和整理的定义：

  总结是一个消化吸收的过程，整理的实质是在归类这些被消化吸收了的知识，以方便用最高效的方式被唤醒。

总结文献的方法：

  1. 预筛：
  • 不是每篇文献都是需要总结的，提前预筛需要总结的文献，通过快速浏览摘要，关键词，结论等部分做文献预筛选，筛选的核心标准就是跟自己研究方向的契合程度。
  1. 阅读
    （1）阅读前：
  • 一定要思考：为什么要总结这篇文章，写下来随时提醒自己。
    （2）阅读中：
  • 初阅读：简单阅读每个章节标题去确定文章的研究问题和研究意义，把握整体脉络；
  • 深度阅读：在交互式阅读中，标注重点，整理文章脉络，画细节，写标注；
  • 吸收：结合之前的标注，用自己的话阐述这篇文章的整体脉络和关键知识点（避免抄袭）如果不能用自己的话阐述，可能还没有完全理解本论文。
    （3）阅读后：
  • 以 Minireview 的形式总结学术最新进展。每篇文章都是一个小的little review, 10-20篇文章的系统梳理，就可以整理成Mini review，每周写一到两篇的评论性文章。

整理文献的方法：

（1）思想：轻整理，重搜索 - 推荐工具使用（Mendeley）；
（2）同一类的文献放在一个文件夹底下，也不要切分太细；
（3）将整理的Minireview 放到文件夹底下；

如何走出颓废时光

发表于 2021-03-04 | 分类于 锦囊

总述：当我进入丧的状态时，我该如何做？


接下来，对上面的Figure 做具体的阐述。

1. 突然进入对什么都不感兴趣，行尸走肉般的状态的原因是什么？

- 没有享受过程

不是没有毅力，懒散，消极 关键在于，你没法从自己所做的每件事情中，获得足够的愉悦，活力 和 成就感。对于每一事物，我们应该学着积极地去经历一个过程，而不是去完成一个确定的结果。以结果为导向，是固定型思维，以过程为导向，是成长型思维。

如何快速走出行尸走肉的状态？

- 放纵心灵，放纵身体，感知社会，开启新知

（1）大力耍废 - 给自己放个小假又如何
（2）大量流汗
（3）大量社交
（4）灌溉领域新职

人生焦虑的原因？

- 渺小力量的自我，妄图掌握过于强大的目标。

如何解决人生焦虑？

- 大事化小，难事儿化易。

做一件事儿，如果目光一直盯着未来那个较长时间尺度的最终目标，就会产生一种匮乏感和无力感。难以把控，甚至最终会对这件事儿失去控制。我们是活在当下的生物，而不是活在未来。将大事化细，分解成自己能力之下可完全掌控的小事儿；将难事儿化易，分解成无数个自己可毫无负担做成的小事儿。

为什么被手机绑架，感觉精力被掏空了？

-  透支人生，精力耗散，追求快感，逃避现状

（1）提前透支了自己人生体会后的空虚，仿佛从虚拟中过完了一生，在回到现实去过，就觉得索然无味。 - 世界上只有一种真正的英雄主义，就是认清了生活的真相后还依然热爱它
（2）注意力被过度耗散后的不专注。信息泛滥让人的持续专注力普遍下降了。微软曾经做过一个调查，2000年人们的持续专注力还有12秒，2015年只有8秒了。那2019怕不是只有5秒了。但真正能够让你的大脑汲取到营养的，都是需要高度且长时间专注的事，比如读书。循序渐进夺回自己的注意力。
（3） 快感刺激阈值被提高。
（4）除了逃避自己的现状外，我们也在利用玩儿手机来拖延、逃避眼前亟待解决的问题，活在有钱人用钱砸出来的虚幻的泡泡。

推动自己放下手机，马上去学习的方法？

- 逼着自己启动

两分钟的自控力应该有吧，让自己停止思考，然后强迫自己的开始工作，只要一开始了工作，后面的事情就容易多了。

慢慢走向正规之后，如何开始做计划？

- WOOP

wish ：提出愿望
Outcome： 想象你完成了这一愿望，会产生什么良好结果
Obstacle：找出有什么障碍会阻止你实现愿望
Plan： 指定时间和地点，思考出一个应付障碍的行为。

如何快速建立一个Rstudio项目并关联到github

发表于 2021-02-11 | 分类于 计算机技术

思路一：

  1. 然后创建远程Repository
  2. 将远程repository 拉到本地
  3. 创建一个本地的Rstudio 项目，将其创建到刚刚拉下来的folder 中。
  4. 在 Rstudio项目中创建相应的文件夹,
    • /input
    • /output
    • /script
    • /figure
    • …….
  1. 关掉Rproj，重新开启，应该就自动弹出git 按钮
  2. 按照git按钮提交更新并push

  1. 创建一个本地的R proj ，将其创建在新的folder底下，此时会弹出 Create project as subdirectory 选项卡
